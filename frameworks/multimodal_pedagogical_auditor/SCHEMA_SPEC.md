# Auditor JSON Schema Specification (v2.0)

This document establishes the strict data contract for the JSON payload generated by the AI Pedagogical Auditor at the conclusion of every session report. This schema ensures deterministic, frictionless database ingestion for downstream automation tools like SyncEngine.

## Core Data Types & Fields

| Key | Type | Description |
| :--- | :--- | :--- |
| `course_code` | String | The unique academic identifier (e.g., "PY101"). |
| `course_name` | String | The formal title of the curriculum (e.g., "Introduction to Python programming"). |
| `course_track` | String | The specialized program or pathway category. |
| `lesson_title` | String | The specific title of the audited lesson. |
| `lesson_theme` | String | The core subject focus or topical anchor for the week. |
| `lesson_sequence` | Integer | The sequential chronological position of the lesson. |
| `duration_string` | String | The human-readable runtime of the session formatted as `MM:SS`. |
| `duration_seconds` | Integer | The absolute runtime converted into a pure integer for temporal data tracking. |
| `timestamp` | String | The precise execution date and time string. |
| `model_version` | String | Locked to `Gemini 3.1 Flash Live`. |
| `metrics` | Object | Nested analytical scoring dictionary. |
| `qualitative_data` | Object | Nested structural summary strings. |
| `mastery_skills_acquired` | Array (Strings) | List of explicit individual technical or theoretical capabilities achieved. |
| `overall_verdict` | String | The final multi-sentence evaluative summary. |

---

## Nested Object Definitions

### 1. Metrics Schema (`metrics`)
All values must return a standardized qualitative rating constraint string: `[High / Optimal / Low]` or `[Pass / Fail]` depending on application thresholds.
*   `cognitive_load`: Conceptual pacing and mental friction analysis.
*   `delivery_precision`: Phrasing accuracy and concept clarity.
*   `domain_integrity`: Technical correctness and documentation adherence.
*   `constraint_adherence`: Verification against declared instructional boundaries.
*   `assessment_integrity`: Evaluation of quiz precision, broken keys, or ambiguity.

### 2. Qualitative Data Schema (`qualitative_data`)
*   `lesson_blueprint`: Summary of Section 1 constraints and structure.
*   `refactoring_notes`: Summary of Section 3 pedagogical alternatives.
*   `systemic_errors`: Summary of Section 4 structural gaps and platform discrepancies.

---

## Strict Zero-Inference Fallback Rule
If a target variable is omitted from the live instructional text, text tags, or user highlights, and cannot be statically found on screen, the field must resolve strictly to `"N/A"` (for strings) or `null` (for integers/objects). AI hallucination or inference of empty metadata properties violates schema validation.
